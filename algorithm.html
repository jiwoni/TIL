컴퓨터 알고리즘 정의 : 효율적으로, 단계적으로 문제를 해결하는 방법
    keyword: 효율성, 단계적, 문제, 해결방법

컴퓨터 알고리즘을 설명하기 위한 4단계
-문제 정의: ex) 해결하고자 하는 문제는 무엇인가?
-알고리즘 설명: 컴퓨터가 수행해야 할 내용은 하나씩 차례대로 정의한 과정
-정확성 증명: ex)과정대로 수행하면 출력으로 항상 올바른 답을 내보내는가? , 잘못된 답을 내보내는 경우는 없는가? , 올바른 출력을 내보내고 정상적으로 죵료되는가?
-성능 분석: 수행시간(Running time), 사용공간(Space consumption)

										
##알고리즘 수행시간 분석

-수행연산의 횟수를 비교하는 방식으로 성능을 분석

ex) 10번 고정(연산 횟수 동일): 공정한 비교
    android 10s => 1s * 10번 
    pc 1s => 0.1s * 10번 
    server 0.1s => 0.01s * 10번
좋은 알고리즘은 연산의 횟수를 줄이는 것


###성능 분석의 비교 대상

-산술 연산(Arithmetic Calculation) : Add, Multiply, Exponent, Modular...
-데이터 입출력 (Data Movement) : Copy, Move, Sava, Load...
-제어 연산(Access Control): if, while, Register....

- 점근적 표기법(Asymptotic notation)
-notation(빅오 표기): $O$$(g(n))$ $=$ $\{f(n)$ : $n$ $\geq$ $n_{0}$ 인 모든 $n$에 대해 0 $\preceq$ $f(n)$ $\preceq$ $cg(n)$을 만족하는 양수 상수 $c$와 $n_{0}$가 존재 $\}$

- $n_{0}$보다 오른쪽에 있는 모든 $n$에 대해서 함수 $f(n)$의 값은 $cg(n)$과 같거나 아래쪽에 있다.
- $g(n)$을 함수 $f(n)$의 점근적 상한(asympotic upper bound)이라고 한다.
- 그림 $f(n)$이 성능 분석의 대상함수, $cg(n)$는 기준이 되는 함수
- 점근적 상한은 시간(y축)이 지남에 따라 아무리 $f(n)$이 커지더라도 $cg(n)$보다는 커지지 않는다.



-notation(오메가 표기): $\Omega $$(g(n))$ $=$ $\{f(n)$ : $n$ $\geq$ $n_{0}$ 인 모든 $n$에 대해 0 $\preceq$ $cg(n)$ $\preceq$ $f(n)$을 만족하는 양수 상수 $c$와 $n_{0}$가 존재 $\}$

- $n_{0}$보다 오른쪽에 있는 모든 $n$에 대해서 함수 $f(n)$의 값은 $cg(n)$과 같거나 위쪽에 있다.
- $g(n)$을 함수 $f(n)$의 점근적 하한(asympotic lower bound)이라고 한다.
- 그림 $f(n)$이 성능 분석의 대상함수, $cg(n)$는 기준이 되는 함수
- 점근적 하한은 시간(y축)이 지남에 따라 $f(n)$이 $cg(n)$보다는 내려갈 수 없는 범위를 찾는 문제(결국 $cg(n)$보다 느리다).



-notation(쎄타 표기): $\Theta $ $(g(n))$ $=$ $\{f(n)$ : $n$ $\geq$ $n_{0}$ 인 모든 $n$에 대해 0 $\preceq$ $c_{1}g(n) $ $\preceq$ $f(n)$ $\preceq$ $c_{2}g(n)$ 을 만족하는 양수 상수 $c_{1}$, $c_{2}$와 $n_{0}$가 존재 $\}$

- $n_{0}$보다 오른쪽에 있는 모든 $n$에 대해서 함수 $f(n)$의 값은 c_{1}g(n) 과 같거나 위쪽에 있고, c_{2}g(n)과 같거나 아래쪽에 있다.
- $g(n)$을 함수 $f(n)$의 점근적 상한 및 하한(asympotic tight bound)이라고 한다.
- 그림 $f(n)$이 성능 분석의 대상함수, $cg(n)$는 기준이 되는 함수
- 점근적 상한 및 하한은 c_{1}g(n) 과 c_{2}g(n) 사이에 있는 함수, 즉 아무리 빨라도 c_{1}g(n) 보다는 느리고 c_{2}g(n) 보다는 빠르다.
